# plan-cycle

## 适用场景
- 新一轮需求进入，需要重排优先级与里程碑。
- 既有计划发生漂移，需要重建阶段边界。
- 周期性复盘后，需要把结论写回 `plan.md`。

## 执行流程
1. 读取 `.dalek/pm/plan.md` 当前内容，确认目标、范围、风险与依赖。
2. 基于最新上下文更新阶段划分（phase）、验收口径与退出条件。
3. 把阶段映射到可执行 ticket（包含顺序、并行关系与阻塞关系）。
4. 记录本轮决策依据与 trade-off，避免口头决策丢失。
5. 写回 `plan.md`，确保文档可被下一轮直接续跑。

## 硬约束
1. 先读后改：未读取当前 `plan.md` 前不得直接覆盖。
2. 决策必须可追踪：关键取舍要写入“决策记录”区块。
3. 每个阶段必须包含验收标准与风险处理策略。
4. 不在此技能里直接改业务代码；只维护计划层文档。

## 输出要求
- `plan.md` 中至少更新：当前阶段、下一阶段、阻塞项、验收标准。
- 形成一段可直接用于调度的执行摘要（供 PM dispatch 使用）。
