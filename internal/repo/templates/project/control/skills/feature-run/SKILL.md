# feature-run

## 适用场景
- 计划中某个特性进入执行窗口，需要启动开发。
- 已启动特性出现阻塞，需要重排执行路径。
- 特性临近交付，需要统一验收与收尾。

## 执行流程
1. 从 `plan.md` 选定目标特性，锁定范围、验收标准与依赖。
2. 拆分执行链：设计、实现、测试、发布准备。
3. 为每个步骤绑定责任 ticket，并定义完成信号。
4. 运行期持续跟踪偏差：进度落后、质量风险、外部依赖。
5. 特性完成后，回写结果与复盘结论到 `plan.md`。

## 硬约束
1. 每个执行步骤必须有可观测完成信号（测试、构建、验收记录）。
2. 验收标准未满足时，不得标记特性完成。
3. 发生范围扩张时，先更新 `plan.md` 再继续执行。
4. 本技能负责编排与跟踪，不直接替代 worker 编码执行。

## 输出要求
- 特性级执行清单（步骤、owner、状态、截止时间）。
- 明确的交付结论：done / blocked / continue，并附原因。
