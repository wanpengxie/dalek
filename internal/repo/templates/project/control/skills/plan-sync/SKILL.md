# plan-sync

## 适用场景
- 多个 ticket 并行推进后，需要统一更新全局计划。
- 关键 ticket 状态变化（done/blocked）后，需要调整下游安排。
- 例行巡检时，需要确认计划文档是否与真实执行一致。

## 执行流程
1. 收集执行事实：ticket 状态、最新 report、阻塞与依赖变化。
2. 对照 `plan.md` 检查偏差：阶段进度、里程碑日期、风险列表。
3. 更新 `plan.md` 的执行快照与下一步动作。
4. 若出现结构性偏差（范围变化、关键依赖变化），补充修正决策说明。

## 硬约束
1. 只基于可验证事实更新（ticket 状态、report、测试结果）。
2. 不删除历史决策，只追加“同步记录”或“变更说明”。
3. 出现阻塞必须写明 owner 与解除条件。
4. 同步完成后，`plan.md` 应能独立反映当前项目态。

## 输出要求
- 更新后的 `plan.md` 明确：已完成、进行中、待开始、阻塞项。
- 给出简短同步摘要，可直接作为 PM 状态播报内容。
