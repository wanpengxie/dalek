# Dalek Web Console PRD

## 背景

当前 dalek 主要通过 CLI / TUI 管理项目、ticket、worker、merge 与 inbox。
下一代 dalek 需要提供一个真实可用的 web 管理页面，让用户可以在浏览器中查看与操作系统状态。

## 目标

- 为 dalek 提供浏览器可访问的管理入口
- 覆盖 PM 最核心的观测与管理场景
- 作为“PM 以真实场景验收需求”的标准产品 case

## 目标用户

- 使用 dalek 管理 AI worker 的技术负责人 / PM
- 需要快速查看系统状态的开发者

## MVP 范围

第一版必须提供：
- 项目概览页
- tickets 列表与详情页
- merge / inbox 状态页
- planner / runtime 状态页
- 至少一个真实操作链路可用并可验证状态变化

第一版不要求：
- 完整权限系统
- 完整视觉设计系统
- 所有 CLI 功能的 web 等价实现

## 用户核心任务

1. 打开页面后快速看到项目总体状态
2. 查看 ticket 当前进度与阻塞
3. 查看 planner 是否在自动推进
4. 查看 merge / inbox 待处理项
5. 至少完成一个真实管理动作，并观察状态变化

## 成功标准

- 用户可以通过浏览器访问 web 管理页面
- 页面能真实展示 dalek 的运行状态，而不是静态 mock
- 至少一个真实用户操作链路可以跑通
- PM 可以用该场景完成真实产品验收

## 验收口径

- `ticket + worker`：以测试 / 构建 / lint 为主
- `PM`：以真实浏览器场景验收为主，测试 / 构建为辅
- 只有真实浏览器链路通过，才能宣布 feature 完成
